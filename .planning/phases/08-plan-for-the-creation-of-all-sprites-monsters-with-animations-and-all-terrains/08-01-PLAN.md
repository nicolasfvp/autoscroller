  <action>
Use PixelLab MCP tools to create 6 monster characters and their animations. All characters must be 64x64 pixels, side view, using the "mannequin" template_id for consistency with the hero.

**Idempotency guard:** Before creating a character or starting an animation job, check if `public/assets/monsters/{id}/metadata.json` already exists (for character creation) or if frame PNGs already exist in the target animation directory (for animation jobs). If assets already exist for a monster, log "Skipping {id} — already generated" and move on. This makes the task restartable without wasting PixelLab credits.

**Step 1 — Create characters (can run in parallel):**

Use `mcp__pixellab__create_character` for each (skip if metadata.json already exists):

1. **slime**: prompt "green slime monster, gelatinous blob creature, simple round body, fantasy RPG enemy", size 64x64, directions 4, view "side", template_id "mannequin"
2. **goblin**: prompt "goblin warrior with dagger, small green-skinned humanoid, leather armor, pointy ears, fantasy RPG enemy", size 64x64, directions 4, view "side", template_id "mannequin"
3. **orc**: prompt "orc brute with club, large muscular green humanoid, crude armor, tusks, fantasy RPG enemy", size 64x64, directions 4, view "side", template_id "mannequin"
4. **mage**: prompt "dark mage in purple robes, hooded spellcaster, glowing staff, mystical energy, fantasy RPG enemy", size 64x64, directions 4, view "side", template_id "mannequin"
5. **elite_knight**: prompt "elite knight in silver armor, full plate mail, large sword, ornate helmet, fantasy RPG elite enemy", size 64x64, directions 4, view "side", template_id "mannequin"
6. **boss_demon**: prompt "demon lord boss, dark red skin, horns, wings, massive muscular body, glowing eyes, fantasy RPG boss", size 64x64, directions 4, view "side", template_id "mannequin"

**Step 2 — Wait for character creation to complete:**
Use `mcp__pixellab__get_character` to check status of each character. Wait until all 6 are complete.

**Step 3 — Add animations to each character (can run in parallel):**

Use `mcp__pixellab__animate_character` for each character (skip if frame PNGs already exist in the target animation directory):
- Animation 1: "breathing-idle" (idle animation)
- Animation 2: "1h-attack" (attack animation — for slime use "jump" instead since slimes don't have weapons)

For boss_demon additionally add: "walking" animation (boss needs a menacing walk cycle).

**Step 4 — Wait for all animations to complete:**
Use `mcp__pixellab__get_character` to check animation status. Each animation is an async job — poll until complete.

**Step 5 — Download all assets:**
For each monster, download:
- The south-east rotation frame to `public/assets/monsters/{id}/rotations/south-east.png`
- All animation frames for breathing-idle/south-east to `public/assets/monsters/{id}/animations/breathing-idle/south-east/`
- All animation frames for attack animation/south-east to `public/assets/monsters/{id}/animations/attack/south-east/`
- Save character metadata to `public/assets/monsters/{id}/metadata.json`

Create directories as needed: `mkdir -p public/assets/monsters/{id}/animations/breathing-idle/south-east` and `mkdir -p public/assets/monsters/{id}/animations/attack/south-east`

NOTE: PixelLab jobs are async and take 2-5 minutes each. Be patient polling status. If a creation fails, retry once with a simplified prompt.

  <action>
Use PixelLab MCP tile creation tools to generate 5 special tile icon sprites. These are 64x64 pixel art images that will overlay on top of terrain tiles in the game world.

Use `mcp__pixellab__create_tiles_pro` or `mcp__pixellab__create_topdown_tileset` (whichever is available) for each:

1. **tile_shop**: prompt "medieval market stall shop icon, gold coins, merchant cart, top-down pixel art RPG tile", 64x64
2. **tile_rest**: prompt "campfire rest site icon, warm fire with logs, cozy camp, top-down pixel art RPG tile", 64x64
3. **tile_event**: prompt "mysterious question mark event icon, glowing scroll, magical encounter, top-down pixel art RPG tile", 64x64
4. **tile_treasure**: prompt "treasure chest icon, golden chest with gems, loot pile, top-down pixel art RPG tile", 64x64
5. **tile_boss**: prompt "skull boss icon, demonic skull with horns, danger marker, top-down pixel art RPG tile", 64x64

If the tile tools produce tilesets rather than individual tiles, extract the best single 64x64 tile from each tileset using sharp:
```
node -e "require('sharp')('output.png').extract({left:0,top:0,width:64,height:64}).toFile('public/assets/tiles/tile_shop.png')"
```

NOTE: Do NOT generate bg_*.png background objects for special tiles. Special tiles (shop, rest, event, treasure, boss) only need the icon overlay sprite. Background objects are only used for terrain tiles (basic, forest, graveyard, swamp) which already have their bg_*.png assets.

If PixelLab tile tools are not suitable for small icons, use `mcp__pixellab__create_character` with a "prop" or item-like prompt as fallback, then crop/resize the output to 64x64.
