# Phase 2 -- UI Design Contract

> Visual and interaction contract for the combat + deck engine phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Phaser 3 canvas rendering) |
| Preset | not applicable |
| Component library | none -- all UI is Phaser GameObjects (Text, Rectangle, Container, Graphics) |
| Icon library | none -- unicode glyphs and geometric shapes (existing pattern from Phase 1) |
| Font | System stack via Phaser: `"Inter, system-ui, Avenir, Helvetica, Arial, sans-serif"` (inherited from `style.css`) |

**Context:** Inherits all conventions from Phase 1 UI-SPEC. Canvas is 800x600. All in-game UI is Phaser GameObjects. This phase introduces 7 new UI surfaces: Combat HUD, Card Queue Display, Synergy Flash, Post-Combat Summary, Card Reward Choice, Deck Viewer, Shop/Deck Editor, and Death Screen.

## Spacing Scale

Inherited from Phase 1. All values are canvas pixels at 800x600 resolution.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, card border insets |
| sm | 8px | Compact element spacing, bar padding, card-to-card gap in queue |
| md | 16px | Default element spacing within panels, card content padding |
| lg | 24px | Section padding inside overlay panels, combat zone margins |
| xl | 32px | Gap between major HUD groups, card queue offset from combat area |
| 2xl | 48px | Panel margin from screen edges |
| 3xl | 64px | Card reward choice spacing between offered cards |

Exceptions: Touch-target minimum for drag-and-drop card items in shop is 44px height (accessibility standard for pointer targets).

## Typography

Inherited from Phase 1 with one addition: Combo text uses Display size for visual punch.

| Role | Size | Weight | Line Height | Phaser Style |
|------|------|--------|-------------|--------------|
| Body | 16px | 400 (regular) | 1.5 | `{ fontSize: '16px', color: '#ffffff' }` |
| Label | 14px | 400 (regular) | 1.4 | `{ fontSize: '14px', color: '#ffffff' }` |
| Heading | 24px | 700 (bold) | 1.2 | `{ fontSize: '24px', fontStyle: 'bold' }` |
| Display | 32px | 700 (bold) | 1.2 | `{ fontSize: '32px', fontStyle: 'bold' }` |

**Phase 2 usage mapping:**
- **Label (14px):** Card cost values, cooldown timer numbers, resource bar labels, stat numbers in summary
- **Body (16px):** Card names in queue, combat log entries, stat labels, deck viewer card text
- **Heading (24px):** Scene titles ("Combat", "Rewards", "Deck"), stat section headers in summary, button labels
- **Display (32px):** "COMBO!" synergy flash text, "VICTORY" / "DEFEAT" text, death screen title, damage numbers

## Component Inventory (Phaser GameObjects)

### Card Visual (reusable across all card-showing scenes)
- Rectangle: 72px wide x 96px tall at `0x222222`
- Stroke: 2px, color based on rarity tier (common/uncommon/rare)
- Type indicator: 4px tall colored strip at top of card (attack=red, defense=blue, spell=purple)
- Card name: Body (16px) centered, white text, word-wrap at 64px
- Cost indicator: Label (14px) bottom-left, stamina color or mana color based on card type
- Cooldown indicator: Label (14px) bottom-right, muted text, shows seconds (e.g., "1.2s")
- Hover state: scale to 1.1x with 200ms ease, stroke brightens to white

### Card Queue Display (Combat Scene - right side)
- Vertical list of Card Visuals, right-aligned at x=728 (800 - 72)
- Shows 5 cards visible at a time (5 x 96px + 4 x 8px gap = 512px)
- Current card (index 0): full opacity, highlighted with accent pulse glow (Phaser tween, 0.8s loop)
- Next cards (index 1-4): 0.7 alpha, no highlight
- Position indicator: small triangle arrow left of current card
- When a card plays: top card slides left and fades (300ms), remaining cards slide up (200ms)
- Skipped card: flash muted text "Skipped" for 600ms, card dims briefly then restores

### Combat HUD (Combat Scene - top area)
- Container at y=8, full width, setScrollFactor(0), setDepth(100)
- **Hero section (left, x=8):**
  - HP bar: 160px wide, 12px tall, semantic HP color fill
  - HP text: Label (14px), "{current}/{max}" right of bar
  - Stamina bar: 120px wide, 8px tall, stamina color fill
  - Stamina text: Label (14px), "{current}/{max}"
  - Mana bar: 120px wide, 8px tall, mana color fill
  - Mana text: Label (14px), "{current}/{max}"
  - Vertical stack with 4px gap between bars
- **Enemy section (right, x=520):**
  - Enemy name: Body (16px), white
  - HP bar: 160px wide, 12px tall, destructive color fill
  - HP text: Label (14px), "{current}/{max}"
- **Cooldown indicator (center, x=400):**
  - Circular arc (Phaser Graphics) showing hero cooldown progress
  - 24px radius, accent color arc fill
  - Center text: remaining seconds at Label (14px)

### Synergy Flash (Combat Scene - center overlay)
- "COMBO!" text at Display size (32px bold), synergy color (`#ff00ff`)
- Position: canvas center (400, 300)
- Animation: scale from 0 to 1.3 over 200ms (Back.easeOut), hold 400ms, fade to 0 over 300ms
- Glow effect: both cards in queue get 4px synergy-colored stroke for 800ms
- Bonus text below: Body (16px), synergy color, shows bonus description (e.g., "+8 damage")
- Bonus text animation: fade in 100ms after COMBO, float up 30px and fade over 600ms

### Post-Combat Summary (Overlay Scene)
- Overlay panel: centered Rectangle 500x380 at `0x222222`, alpha 0.9
- Title: Heading (24px bold), "Battle Summary", centered at top with 24px padding
- Stats grid: two columns, left-aligned labels (Body 16px, muted) + right-aligned values (Body 16px, white)
  - Damage Dealt: value in white
  - Damage Taken: value in destructive color if > 0
  - Cards Played: value in white
  - Combos: value in synergy color if > 0
  - XP Earned: value in XP color
- Vertical spacing: 8px between stat rows, 24px between stats section and CTA
- "Continue" button: Heading (24px bold), accent color, centered at bottom with 24px padding

### Card Reward Choice (Overlay Scene)
- Overlay panel: centered Rectangle 600x400 at `0x222222`, alpha 0.9
- Title: Heading (24px bold), "Choose a Card", centered top
- Instruction: Body (16px), muted, below title with 8px gap
- Three Card Visuals at enlarged size (108px x 144px): horizontally centered with 64px gap between them
  - Cards positioned at y=200 (vertical center of panel)
  - Each card is interactive (pointer cursor)
  - Selected card: accent stroke (3px, `#ffd700`), scale 1.05
  - Unselected cards: dim to 0.5 alpha when one is selected
- "Take Card" button: Heading (24px bold), accent color, below cards, visible only when a card is selected
- "Skip" button: Body (16px), muted color, bottom-right corner of panel

### Deck Viewer (Overlay Scene)
- Overlay panel: centered Rectangle 600x450 at `0x222222`, alpha 0.9
- Title: Heading (24px bold), "Your Deck", top-left with 24px padding
- Card count: Label (14px), muted, right of title
- Card grid: 6 columns x scrollable rows, Card Visuals at standard size (72x96)
  - 8px gap between cards
  - Grid starts at y=64 within panel (below title)
  - If > 18 cards (3 rows), enable vertical scroll via drag-to-scroll on the grid container
- Synergy pairs highlighted: cards that form a synergy pair with an adjacent card get a subtle synergy-colored bottom border (2px)
- "Close" button: Body (16px), muted, top-right corner

### Shop Deck Editor (Overlay Scene)
- Overlay panel: centered Rectangle 640x460 at `0x222222`, alpha 0.9
- Title: Heading (24px bold), "Reorder Deck", top-left with 24px padding
- **Reorder mode (inactive):**
  - Card list: vertical stack of card items, each 640px wide x 48px tall
  - Card item: type indicator (8px colored circle, left), card name (Body 16px), cooldown (Label 14px, right), cost (Label 14px, muted, right)
  - "Start Reorder (30 Gold)" button: Heading (24px bold), accent color
- **Reorder mode (active):**
  - Same card list but each item has drag handle (unicode `:::` at left, 16px, muted)
  - Cards are draggable vertically (Phaser `setInteractive({ draggable: true })`)
  - Drop target: 4px accent-colored line appears between cards on drag-over
  - Active label: "Drag cards to reorder." (Body 16px, accent color, top)
  - "Done Reordering" button replaces start button
- **Remove mode:**
  - Each card item has "Remove" button (Label 14px, destructive color) at far right
  - Cost shown in parentheses next to button
  - Clicking triggers confirmation overlay (secondary panel 300x160, centered)

### Death Screen (Full Scene)
- Full scene background: dominant color (`#1a1a2e`)
- Title: "Run Over" at Display (32px bold), destructive color, centered at y=80
- Cause: "Defeated by {enemyName}" at Body (16px), muted, centered at y=120

    - Total Damage Dealt: Heading (24px) value
    - Total Cards Played: Heading (24px) value
    - Total Combos: Heading (24px) value, synergy color
  - Labels in Body (16px), muted, left-aligned
  - Values in Heading (24px), white, right-aligned
- XP warning: "All unbanked XP has been lost." at Body (16px), destructive color, below stats panel with 16px gap
- "Return to Menu" button: Heading (24px bold), accent color, centered at y=520

## Interaction Patterns

### Combat Flow (no player input during combat)
1. CombatScene creates, CombatEngine starts tick loop
2. Cards play automatically from queue -- CardQueueDisplay animates each play
3. Synergy triggers show SynergyFlash overlay
4. Enemy attacks show damage particles + floating numbers (reuse CombatEffects)
5. On victory: "VICTORY" display text, 1s hold, transition to PostCombatScene
6. On defeat: "DEFEAT" display text, 1s hold, transition to DeathScene

### Card Queue Watching
- Player watches the vertical card queue on the right side during combat
- Current card pulses with accent glow to draw attention
- Cards slide up as each resolves, giving a "conveyor belt" feel
- Player can hover any card in queue to see a tooltip with full card details (appears as a secondary panel left of the queue, 200ms delay)

### Card Reward Interaction
1. Three cards appear with 300ms staggered entrance (slide up + fade in)
2. Player clicks a card to select it (accent highlight)
3. Clicking another card switches selection
4. "Take Card" confirms choice and closes scene
5. "Skip" closes scene without adding a card
6. If no reward triggered (empty state), show message + auto-continue after 2s or on click

### Deck Reorder Drag-and-Drop
1. Player clicks "Start Reorder" and pays gold
2. Card list enters drag mode (drag handles appear)
3. Player drags a card vertically to new position
4. Drop indicator (accent line) shows where card will land
5. On drop: list reorders, synergy highlights update immediately
6. Player clicks "Done Reordering" to confirm and exit mode

### Scene Transitions (inherits Phase 1 pattern)
- All scenes read from RunState via `getRun()` in `create()`
- No data passing via `scene.launch('SceneName', { data })`
- Combat end writes only HP back to RunState
- Post-combat summary reads CombatStats from CombatEngine.getStats()

## Phase-Specific Notes

Phase 2 is the **core gameplay UI phase**. Every visual element defined here supports the game's central thesis: the player is an architect, not a fighter. The combat screen is deliberately hands-off (no buttons to press during combat), and all strategic interaction happens in the deck editor and reward selection.

Key design principles for this phase:
1. **Card Queue is king.** The right-side queue is the primary visual element during combat. It must be readable, animated smoothly, and clearly show which card is next.
2. **Synergy feedback must be unmissable.** The "COMBO!" flash at center screen + card glow ensures players notice and learn synergy pairs.
3. **Type colors are consistent everywhere.** Attack=red, Defense=blue, Spell=purple appears on every card visual in every scene.
4. **Resource bars are always visible.** HP/Stamina/Mana bars in the combat HUD let the player understand WHY cards get skipped (resource scarcity).
5. **Drag-and-drop is the primary strategic input.** The shop deck editor must feel responsive -- drag handles, drop indicators, and immediate synergy highlight updates.
