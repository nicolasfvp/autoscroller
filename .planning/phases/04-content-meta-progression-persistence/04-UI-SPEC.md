# Phase 4 -- UI Design Contract

> Visual and interaction contract for the content, meta-progression, and persistence phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Phaser 3 canvas rendering) |
| Preset | not applicable |
| Component library | none -- all UI is Phaser GameObjects (Text, Rectangle, Container, Graphics) |
| Icon library | none -- unicode glyphs and geometric shapes (inherited from Phases 1-3) |
| Font | System stack via Phaser: `"Inter, system-ui, Avenir, Helvetica, Arial, sans-serif"` (inherited from `style.css`) |

**Context:** Inherits all conventions from Phase 1, 2, and 3 UI-SPECs. Canvas is 800x600. All in-game UI is Phaser GameObjects. This phase introduces 6 new UI surfaces: City Hub Scene, Building Interaction Panel, Collection Screen, Tavern Panel (run start + seed input + run history), Relic Tooltip (in-run), and Run End Meta-Loot Summary.

## Typography

Inherited from Phases 1-3. No new roles needed.

- **Display (32px):** "Welcome Back" hub greeting, "Run Complete" safe exit summary title, meta-loot earned total, "New Unlock!" celebration text

## Interaction Patterns

### City Hub Navigation
1. City Hub scene loads after game boot (if no active run) or after run end ("Return to City")
2. Player sees 5 buildings arranged in cross pattern around center
3. Hovering a building shows its name and tier
4. Clicking a building opens Building Interaction Panel as overlay
5. Player can click "Collection" button to open Collection Screen
6. Clicking the Tavern opens the Tavern Panel (special variant with run start)

### Building Upgrade Flow
1. Player clicks a building to open its interaction panel
2. Panel shows current tier, progress bar, and upgrade preview
3. Next tier's unlock items are listed with full details
4. Player clicks "Upgrade Building" if they have enough meta-loot
5. Meta-loot is deducted, tier increments, progress bar fills
6. Unlock Celebration overlay plays with each newly unlocked item name
7. Panel refreshes to show new state (next tier preview or "Fully Upgraded")
8. Close panel by clicking backdrop or "X"

### Tavern / Run Start Flow
1. Player clicks Tavern building
2. Tavern Panel opens with "Start Run" button and optional seed input
3. Player optionally enters a seed string (keyboard input via Phaser DOM element)
4. Player clicks "Start Run"
5. Transition to Game Scene (Phase 3 loop world) with seed applied to RNG
6. If seed is blank, random seed is generated and displayed in HUD

### Collection Browsing
1. Player clicks "Collection" from City Hub
2. Collection Screen opens with Cards tab active by default
3. Player clicks tabs to switch categories
4. Grid shows unlocked items with full details, locked items as silhouettes
5. Hovering unlocked items shows full tooltip
6. Locked items show unlock hint ("Unlock via Forge Lv.2")
7. Player clicks "X" or presses Escape to return to City Hub

### Relic Interaction (in-run)
1. Relics appear in HUD strip during run (from Phase 3 Game Scene)
2. Hovering a relic shows Relic Tooltip with name, effect, and source
3. Relic effects are passive -- no click interaction needed
4. New relic acquisition (from drops, shop, events): relic slides into HUD strip with 300ms ease-in, brief accent glow (800ms)

### Run End Transition
1. On death or safe exit: existing Death Screen / Boss Exit scene loads
2. Meta-loot summary rows are appended to existing stats
3. If any new building upgrades are now affordable: "New unlocks available!" notification pulses
4. "Return to City" CTA transitions to City Hub scene
5. City Hub loads fresh with updated meta-loot balance and tier states

### Seed Copy (in-run)
1. Seed string displayed in bottom-right of HUD (muted, low alpha)
2. Player clicks the seed text
3. Seed is copied to clipboard via `navigator.clipboard.writeText()`
4. "Seed copied!" toast appears and fades (1.5s)

### Scene Transitions (inherits Phase 1/2/3 pattern)
- All scenes read from MetaState (new) or RunState via getters in `create()`
- City Hub reads MetaState for building tiers, unlocks, meta-loot balance
- Collection Screen reads MetaState for unlock state
- No data passing via `scene.launch('SceneName', { data })`
- City Hub is a full scene replacement (not overlay)
- Building Interaction Panel is an overlay scene on City Hub

## Phase-Specific Notes

Phase 4 introduces the **second game loop**: between runs, the player is a city builder investing meta-loot to permanently expand the content pool. This creates a meaningful connection between runs -- every run earns meta-loot that shapes future runs.

Key design principles for this phase:

1. **The city IS the meta-game.** The top-down building layout must feel like a place, not a menu. Buildings have distinct colors, positions, and personality. Even with placeholder graphics, spatial arrangement creates identity.
2. **Full upgrade transparency.** Per the locked decision in CONTEXT.md, players see exactly what each building tier unlocks before spending. The upgrade preview list is the most important element in the Building Interaction Panel. No surprise purchases.
3. **Silhouettes create pull.** Locked items in the Collection Screen use `0x444444` fill with "???" names and unlock hints. This is the completionist hook -- seeing what you do not have motivates play.
4. **Two progression paths are visually distinct.** Building upgrades use meta-loot (purple `#e040fb`). Class XP uses progression blue (`#00ccff`). These colors never cross contexts.
5. **The Tavern is the gateway.** It is the central building for a reason -- starting a run is the primary action. Seed input is optional and secondary. Run history provides motivation to beat personal bests.
6. **Relics are visible, passive, and hoverable.** The HUD relic strip lets players see their passive bonuses at a glance without cluttering the existing HUD layout. Tooltips provide full details on demand.
7. **Run end connects to the city.** The meta-loot summary rows and "Return to City" CTA create a direct emotional link: "that run earned me enough to upgrade the Forge."
