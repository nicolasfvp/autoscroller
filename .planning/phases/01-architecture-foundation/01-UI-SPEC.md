# Phase 1 -- UI Design Contract

> Visual and interaction contract for the architecture foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Phaser 3 canvas rendering) |
| Preset | not applicable |
| Component library | none -- all UI is Phaser GameObjects (Text, Rectangle, Container) |
| Icon library | none -- unicode glyphs for HUD icons (existing pattern) |
| Font | System stack via Phaser: `"Inter, system-ui, Avenir, Helvetica, Arial, sans-serif"` (inherited from existing `style.css`) |

**Context:** This is a Phaser 3 canvas game (800x600). All in-game UI is rendered on the canvas via `Phaser.GameObjects`. The only DOM element is the `#app` container that hosts the Phaser canvas. shadcn/tailwind/React are not applicable.

---

## Spacing Scale

Declared values (must be multiples of 4). All values are in canvas pixels at 800x600 resolution.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline text gaps, icon-to-label spacing |
| sm | 8px | Compact HUD element spacing, bar padding |
| md | 16px | Default element spacing within panels |
| lg | 24px | Section padding inside overlay panels |
| xl | 32px | Gap between major HUD groups |
| 2xl | 48px | Panel margin from screen edges |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: none

**Source:** Existing HUDManager.ts uses 10px origin offset and ~5px internal gaps. Standardized to 8-point scale for Phase 1 rewrite.

---

## Typography

All sizes are Phaser `fontSize` values rendered on the 800x600 canvas.

| Role | Size | Weight | Line Height | Phaser Style |
|------|------|--------|-------------|--------------|
| Body | 16px | 400 (regular) | 1.5 | `{ fontSize: '16px', color: '#ffffff' }` |
| Label | 14px | 400 (regular) | 1.4 | `{ fontSize: '14px', color: '#ffffff' }` |
| Heading | 24px | 700 (bold) | 1.2 | `{ fontSize: '24px', fontStyle: 'bold' }` |
| Display | 32px | 700 (bold) | 1.2 | `{ fontSize: '32px', fontStyle: 'bold' }` |

**Notes:**
- Phaser Text does not support `font-weight` numerically. Use `fontStyle: 'bold'` for weight 700.
- All text uses the system font stack declared in `style.css` (inherited by canvas).
- Existing codebase uses 12px-48px range inconsistently. This contract standardizes to 4 sizes.

## Color

All colors expressed as both hex (for Phaser Text `color` property) and Phaser hex integer (for `setFillStyle` / Rectangle).

| Role | Hex | Phaser Int | Usage |
|------|-----|------------|-------|
| Dominant (60%) | `#1a1a2e` | `0x1a1a2e` | Scene backgrounds, full-screen underlays |
| Secondary (30%) | `#222222` | `0x222222` | Panels, HUD backgrounds, overlay containers (with 0.85 alpha) |
| Accent (10%) | `#ffd700` | `0xffd700` | Gold/currency display, primary CTA buttons, save indicator |
| Text Primary | `#ffffff` | n/a | All body text, labels, headings |
| Text Muted | `#aaaaaa` | n/a | Secondary information, disabled states |
| HP High | `#00ff00` | `0x00ff00` | HP bar above 50% |
| HP Medium | `#ffaa00` | `0xffaa00` | HP bar 25%-50% |
| HP Low / Destructive | `#ff0000` | `0xff0000` | HP bar below 25%, destructive actions, damage numbers |
| Heal | `#00ff00` | `0x00ff00` | Healing effects, positive numbers |
| Shield / Defense | `#4169e1` | `0x4169e1` | Defense indicators, shield effects |

Accent reserved for: gold/currency text, gold icon glyph, save-confirmed indicator, primary action button fill on scene transitions (e.g., "Continue" button background).

**Source:** Colors extracted from existing codebase (HUDManager.ts, CombatScene.ts, CombatEffects.ts, DeathScene.ts). Dominant color changed from pure black (`#000000`) to dark navy (`#1a1a2e`) for softer contrast on long sessions. All semantic colors preserved from existing patterns.

## Copywriting Contract

Phase 1 is an architecture rewrite. The only user-facing UI changes are scene transitions, save/load feedback, and the minimal HUD. Copywriting is minimal but must be defined for the thin scene wrappers.

| Element | Copy |
|---------|------|
| Primary CTA | "Continue Run" (when saved run exists on boot) |
| New game CTA | "New Run" (when no saved run exists, or from main menu) |
| Empty state heading | "No Saved Run" |
| Empty state body | "Start a new run to begin your journey." |
| Save indicator | "Saved" (briefly visible after auto-save, 1.5s fade) |
| Save failure | "Save failed. Check browser storage settings." |
| Error state | "Something went wrong. Returning to main menu." |
| Destructive confirmation | Delete Save: "This will permanently erase your current run. Continue?" |
| Confirm button | "Yes, Delete" |
| Cancel button | "Cancel" |

**Language:** English. The GDD is in pt-br but no localization system exists yet. All in-game copy is English for v1.

## Component Inventory (Phaser GameObjects)

Phase 1 reuses and standardizes existing Phaser UI patterns. No new visual components are introduced -- the focus is on architecture. These conventions apply to all thin scene rewrites.

### HUD Panel
- Container with `setScrollFactor(0)` and `setDepth(100)`
- Background: Rectangle at `0x222222` with alpha `0.85`
- Origin: top-left, offset `(8, 8)` from screen edge
- Content: gold display, loop counter, HP bar

### Overlay Panel (for scenes like Shop, Rest, Event, etc.)
- Centered Rectangle at `0x222222` with alpha `0.9`
- Size: max 600x400 within 800x600 canvas
- Padding: 24px internal
- Title text: Display size (32px bold), centered at top
- `setInteractive()` to block click-through

### Button (Text-based)
- Phaser Text with `setInteractive()` and `{ useHandCursor: true }`
- Default state: accent color (`#ffd700`) text
- Hover state: white text (`#ffffff`)
- Disabled state: muted text (`#aaaaaa`), no `setInteractive()`
- Size: Heading (24px bold) for primary actions, Body (16px) for secondary

### HP Bar
- Rectangle filled with semantic HP color (green/orange/red based on percentage)
- Width proportional to current/max ratio
- Max width: 160px
- Height: 12px
- Adjacent label text: `"{current}/{max}"` at Label size (14px)

### Save Indicator
- Phaser Text "Saved" at Label size (14px), accent color
- Position: bottom-right corner, offset `(-16, -16)` from `(800, 600)`
- Fades in over 200ms, holds 1.5s, fades out over 500ms
- `setScrollFactor(0)` and `setDepth(200)` (above all other UI)

## Interaction Patterns

### Scene Transitions
- All scene transitions go through RunState (scenes read `getRun()` in `create()`)
- No data passing via `scene.launch('SceneName', { data })` -- scenes pull from RunState
- Transition: `scene.start('TargetScene')` for full transitions, `scene.launch('OverlayScene')` for overlays

### Auto-Save Feedback
- Save triggers: after combat ends, after shop exit, after boss defeat, after loop completion
- Visual feedback: Save Indicator appears briefly (see component above)
- Failure handling: console.error + persistent warning text at top of screen

### Boot Flow (Load vs New)
1. Boot scene loads all static JSON data via `loadAllData()`
2. Preloader attempts `saveManager.load()`
3. If saved run exists: Main Menu shows "Continue Run" (primary) + "New Run" (secondary)
4. If no saved run: Main Menu shows "New Run" only
5. "New Run" when save exists: shows destructive confirmation dialog

## Phase-Specific Notes

Phase 1 is a **pure architecture phase**. The UI contract here defines conventions that all future phases must follow when building thin scene wrappers. The primary visual deliverables are:

1. **Standardized HUD** rebuilt using the spacing/typography/color tokens above
2. **Save indicator** as the only new visual element
3. **Boot flow** with Continue/New Run logic
4. **Overlay panel template** used by all future scene rewrites

No new gameplay UI is introduced. Combat UI, deck management UI, shop UI, etc. are Phase 2+ concerns and will have their own UI-SPEC contracts.
